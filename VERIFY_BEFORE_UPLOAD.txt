Before uploading to GitHub:

1. Open package.json.
   The first character must be {.
   If it starts with PORT=3010 or TSN_API_BASE_URL, it is wrong.

2. Render commands:
   Build Command: bash ./render-build.sh
   Start Command: node server.js

3. Required behavior:
   - TSN-S opens to a login screen only.
   - Only username ceo can log in.
   - Other usernames show access denied.
   - Wallet is gone.
   - Buy/sell stock is gone.
   - Dashboard shows price, graph, online users, messages/hour, and global chats/hour.

4. Required env vars:
   TSN_API_BASE_URL=https://your-normal-tsn.onrender.com
   TSN_STOCK_ALLOWED_USERNAME=ceo
   TSN_STOCK_SESSION_SECRET=long-random-secret
   MONGODB_URI=your-tsn-s-history-mongodb-uri
   MONGODB_DATABASE=tsn_stock
   MONGODB_COLLECTION=stockSnapshots

5. Deploy with:
   Manual Deploy -> Clear build cache & deploy

v1.2.4 always-on update verification:
1. Deploy TSN-S.
2. Open /healthz and check backgroundUpdater.enabled is true.
3. Open /api/cron/tick?secret=YOUR_SECRET and check ok=true.
4. Log in as ceo and check the alerts panel does not show stale background data.
5. Add a cron/uptime ping if using Render Free.
